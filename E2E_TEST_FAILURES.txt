ISSUE #1: MISSING SERVER
───────────────────────
DESCRIPTION:
The CI/CD workflow builds the Next.js application but never starts a development
or production server before running Playwright tests. This causes the BASE_URL
(http://localhost:3000) to be unreachable, resulting in timeouts and missing
DOM elements.

AFFECTED TESTS:
- All 42 failing tests in article.spec.ts and navigation.spec.ts
- Tests timeout waiting for elements that never render because the app isn't running

ERROR MESSAGES:
- "element(s) not found" timeouts
- Playwright cannot find [role="article"][aria-label^="Read article"]
- Cannot find h1, nav, and other page elements

HOW TO FIX:
1. In .github/workflows/ci.yml, add these steps AFTER the "Build" step:

   - name: Start server
     run: npm run start &
     env:
       NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
       NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}

   - name: Wait for server
     run: npx wait-on http://localhost:3000 --timeout 30000

2. Verify your package.json has a "start" script (for production build):
   "start": "next start"

ISSUE #2: PLAYWRIGHT STRICT MODE VIOLATIONS - MULTIPLE ELEMENT MATCHES
────────────────────────────────────────────────────────────────────────
DESCRIPTION:
Playwright strict mode requires locators to match exactly ONE element. Generic
selectors like 'h1' or 'nav[aria-label="Breadcrumb"]' match MULTIPLE elements
on the page, causing strict mode to fail.

AFFECTED TESTS:
- navigation.spec.ts line 7: locator('h1') → resolved to 2 elements
- navigation.spec.ts line 19: nav[aria-label="Breadcrumb"] → resolved to 2 elements
- navigation.spec.ts line 34: locator('h1') → resolved to 2 elements
- navigation.spec.ts line 39: locator('h1') → resolved to 2 elements
- navigation.spec.ts line 44: locator('h1') → resolved to 2 elements
- navigation.spec.ts line 45: locator('h1') → resolved to 2 elements
- navigation.spec.ts line 46: locator('h1') → resolved to 2 elements
- navigation.spec.ts line 50: locator('h1') → resolved to 2 elements

ERROR MESSAGE EXAMPLE:
Error: expect(locator).toBeVisible() failed
Locator: locator('h1')
Expected: visible
Error: strict mode violation: locator('h1') resolved to 2 elements:
  1) <h1 class="...">Satyamev Jayate</h1>
  2) <h1 class="...">Satyamev Jayate</h1>

HOW TO FIX:
Replace all generic h1 selectors with role-based selectors that specify which h1.

IN frontend/e2e/navigation.spec.ts:

Line 7: Replace:
  await expect(page.locator('h1')).toBeVisible();
With:
  await expect(page.getByRole('heading', { level: 1 }).first()).toBeVisible();

Line 14: Replace:
  await expect(page.locator('h1')).toContainText('Tech');
With:
  await expect(page.getByRole('heading', { level: 1 }).first()).toContainText('Tech');

Line 19: Replace:
  await expect(page.locator('nav[aria-label="Breadcrumb"]')).toBeVisible();
With:
  await expect(page.getByRole('navigation', { name: 'Breadcrumb' }).first()).toBeVisible();

Lines 29, 34, 39, 44, 45, 46, 50: Replace all:
  await expect(page.locator('h1')).toBeVisible();
With:
  await expect(page.getByRole('heading', { level: 1 }).first()).toBeVisible();
